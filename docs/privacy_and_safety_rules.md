# Privacy And Safety Rules

1. Real memory data should stay out of Git by default.
2. Real project content should stay out of Git by default.
3. Sensitive fields must be sanitized.
4. Run the sensitive information scan before pushing.
5. If you are unsure whether a file is safe to publish, do not commit it.

## Additional Rules

- Generated context files should remain local by default.
- Keep only placeholder files from log directories in the repository.
- All examples must be sanitized.
